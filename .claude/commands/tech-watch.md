description: 憲章に沿う新技術を調べて提案する
---
`docs/CHARTER.md` を判断軸に、スコープ内ドメインの新ライブラリ/仕様/手法を調べて提案します。

手順:
1. `docs/CHARTER.md` を読み、Mission / 設計原則 / スコープ内ドメイン / 採用基準を把握。
2. `$ARGUMENTS` にドメイン指定があればそこを、無ければエージェント基盤/A2A/MCP/マルチLLM/
   安全性/観測性/決済を横断して WebSearch/WebFetch で最近の動きを調べる。
3. 各候補を採用基準5項目で評価（目的適合/原則非違反/代替優位/導入コスト/非重複）。
4. `gh issue list --label tech-watch --state open` で重複確認。
5. 基準を満たすものだけ、Issue ドラフト（憲章適合度と導入スケッチ付き）を提示し、確認後に起票。

憲章に反するもの・単一ベンダロックインは提案しないこと。
